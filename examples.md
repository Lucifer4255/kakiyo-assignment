# Test Scenarios

Copy-paste inputs for exercising the app across different outreach angles. Each scenario lists the **prospect input**, an **offering** and a **prompt** to create, and **what good output looks like** (real outputs generated by the pipeline). Reply texts are included so you can test follow-up continuity.

> Tip: enrichment needs real, public sources to produce specific hooks. Use a real public GitHub profile and/or a real company/personal URL. LinkedIn needs uploaded screenshots.

---

## 1. Hiring — recruit an engineer  *(you've already tested this)*

**Prospect input:** a GitHub profile URL + 2–3 LinkedIn screenshots of an engineer who ships AI/agent infra.

**Offering** (name: `Kakiyo — hiring`):
```
We're hiring a Backend/AI Engineer at Kakiyo to own the agent reasoning stack
end to end — prompt orchestration, concurrency, and safety — with no
template-driven workflows. Small senior team, remote-friendly, competitive comp
plus meaningful equity.
```

**Prompt** (name: `Founder hiring`):
```
You are a startup founder personally hiring a strong engineer. You are recruiting
them — not networking, not comparing notes as a peer. Open with one concrete
detail about their actual work. State the role plainly and why they specifically
fit. End with one clear, low-pressure invitation to talk. Four short lines max.
Never end with a technical or curiosity question.
```

**Good output looks like:**
> Ayush, your concurrency-safe agent runtime is impressive.
> We're hiring a Backend/AI Engineer at Kakiyo to own prompt orchestration, concurrency, and safety end-to-end—no templates. Remote, competitive comp + meaningful equity.
> Your experience building agent platforms with MCP orchestration fits this perfectly.
> Would you be open to a quick call?

**Reply test** — paste this as the prospect's reply, then "Draft follow-up":
```
Thanks for reaching out! I'm pretty happy where I am, but I'm curious — what's
the comp range, and is this remote?
```
> Good follow-up (note: stays in voice, answers remote, says comp is competitive **without inventing a number**):
> Glad you're happy where you are. Yes, this is remote-friendly—team is mostly in India time zones. Comp is competitive; I can share the specifics on a call. Worth a quick chat?

## 2. B2B SaaS sales — sell Kakiyo to a growth leader

**Prospect input:** a LinkedIn screenshot of a Head of Growth / VP Sales at a Series A SaaS, plus their company's website URL (Firecrawl scrapes it). GitHub optional/none for a GTM persona.

**Offering** (name: `Kakiyo — product`):
```
Kakiyo is an AI cold-outreach platform. It researches each prospect (GitHub, any
web URL, LinkedIn screenshots) into a structured profile, then writes
hyper-personalized first messages and natural reply follow-ups. A small team can
send outbound that reads 1:1 — without hiring more SDRs.
```

**Prompt:** use the **default** prompt (short, specific, one CTA, under 150 words).

**Good output looks like:**
> Sarah, saw your post about how long SDR ramp is taking — and that you're hiring two right now. That's got to be a tough bottleneck post-Series A.
>
> We built Kakiyo to solve the personalization-at-scale piece. It researches each prospect (GitHub, web, LinkedIn) and writes 1:1 messages and follow-ups. Small teams can send outbound that actually feels human, without burning reps' time on research.
>
> Worth a quick look? I'd love to show you how it fits Northwind's outbound motion.

**Reply tests** (try each — they should steer the follow-up differently):
```
Interesting. How is this different from Apollo or Outreach? We already pay for those.
```
```
We're not really looking to add tools right now, budgets are tight this quarter.
```

## 3. Developer-tool sales — GitHub-only input

Tests **flexible input**: a prospect enriched from *only* a GitHub profile (no LinkedIn, no URL).

**Prospect input:** just a public GitHub profile URL of an active OSS developer.

**Offering** (name: `DevTool — pitch`):
```
We make a CI tool that auto-generates and maintains integration tests from your
existing API traffic, so teams ship faster without writing brittle test suites by
hand. Drops into GitHub Actions in minutes. Free for open source.
```

**Prompt:** default prompt.

**What to check:** the message should hook on something specific from their repos/languages (e.g. a project or stack), proving GitHub-only enrichment still yields a personal angle — not a generic blast.

## 4. Customization check — same prospect, two prompts

Tests the **"prompt visibly changes output"** criterion. Use the **Scenario 2** prospect + `Kakiyo — product` offering, and generate once with each prompt.

**Prompt A** — default (professional, structured).

**Prompt B** (name: `Casual peer`):
```
Write like a peer reaching out, not a sales rep. Two sentences, conversational and
warm, lowercase is fine. Reference one specific thing about them. End with a light,
genuinely low-pressure ask — no hard CTA.
```

**Prompt B output looks like** (clearly different register from A):
> saw your post about sdr ramp taking forever — that's the exact pain we built kakiyo to fix. we help small teams scale 1:1 outbound without needing more headcount; if you're curious, happy to share how it could work alongside your new sdr hires.
